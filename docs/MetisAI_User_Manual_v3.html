멀티테넌트 AgentOps 거버넌스 · 성과(ROI) 플랫폼 — v3 현행화(2026-06)

1. 개요

METIS.AI는 기업이 운영하는 AI 에이전트를 안전하게 통제(거버넌스)하고 그 성과(공수 절감·ROI)를 증명하는 멀티테넌트 SaaS입니다. 모든 에이전트 실행은 자동으로 4게이트 품질 평가·보안 점검·이상 탐지를 거치며, 결과는 대시보드·효과성·리스크 화면에 집계됩니다.

구성내용
핵심 가치① 품질·보안·정책 자동 평가(4게이트) ② 성과/ROI 정량화(시간절감·MTTD·MTTR) ③ 운영 리스크·이상 탐지 ④ FinOps 비용 최적화 ⑤ 지식·정책 자동 학습 루프
구조main agent = 워크플로우, sub agent = 노드. 실행=ExecutionSession/Step, 평가=AgentEvaluation.
대시보드 테마다크 모드(전문가용 저채도 팔레트)

3. 홈 대시보드

메뉴: 대시보드(/home)

3.1 상단 요약 · 빠른실행

상단 기간 선택(7/30/90/180일) 왼쪽에 신규 빠른실행 버튼(최근 사용 1개 · 인기 2개)이 있어 자주 쓰는 에이전트를 바로 실행할 수 있습니다.

요약 카드 6종(클릭 시 상세 팝업/이동):

카드의미
Agent 총 수행기간 내 총 실행 건수 (클릭 → 실행 이력)
수행 성공률성공 비율 %
평균 속도평균 지연(ms)
토큰 비용기간 토큰 비용($)
이상 감지이상 탐지 건수
활용 시스템 신규활용 중인 시스템 수. 클릭 → 시스템별 사용 Agent 팝업(어느 시스템에서 어떤 에이전트가 얼마나 쓰이는지)

3.2 효과성 요약

카드내용
총 절감 공수 변경MM(맨먼스, 1MM=160h) 단위로 표시. 부제는 실제 적용 Agent 수·순가치($). 클릭 → Agent별 절감 효과 팝업(절감 공수 MM·절감률·순가치)
ROI투자 대비 효과(배수). 금액은 천단위 정수 표기
품질 평균 증감이전 기간 대비 품질 증감. 클릭 → Agent별 품질 증감 팝업

3.3 활용 랭킹

활용 Top3 / 미활용 Top3. 신규 에이전트 행 클릭 → 사용 상세 팝업(어느 시스템에서·얼마나·성공률·점수·절감 공수).

3.4 성과·품질 추이

성과 추이(막대=실행량, 선=성공률%), 품질 추이(선=종합점수, 막대=이상). 신규 가로축=날짜, 세로축=지표(0~100) 라벨 표시. "상세" 버튼 → 드릴다운 팝업, "분석 페이지" → 효과성/Evaluator 화면.

3.5 APM X-View 신규

제니퍼 스타일 산점도: 가로축=하루 24시간(시각), 세로축=호출 소요시간(ms), 점=개별 수행(성공/실패/실행중 색상). 마우스로 영역을 드래그하면 선택된 수행들의 상세(건수·평균소요·실패수·주요 Agent·목록) 팝업이 열립니다.

3.6 Agent 현황 · 런처

Agent 현황(Main Agent별): 변경 상위 10개가 한눈에 보이고 나머지는 스크롤로 확인(행 간격 압축). 막대=평균점수·색=상태, 우측에 품질 추이·실행수·이상. 행 클릭 → Sub Agent 상세 + 효과성 블록. 하단에 Agent 런처(실행)·최근 사용. 모든 에이전트는 코드 · 이름(예: DEV-003 · Dev Agent) 형식으로 일관 표기됩니다.

6. 성과/효과성 신규/강화

메뉴: 거버넌스 > 성과/효과성

에이전트·시스템별 효과성을 3단(요약/에이전트별/시스템별)으로 제공합니다.

지표설명
시간 절감 / ROI수작업 기준 대비 절감 시간·% , 순가치(실측)
MTTD평균 탐지 시간(원천신호 수집 시 실측, 없으면 평균 지연 근사 — 출처 표기)
MTTR평균 복구 시간(FDSAlert 생성→해결 타임스탬프 실측)
커버리지테스트 커버리지(원천신호 수집 시 실측, 없으면 목표값)
시스템별 롤업시스템 단위 활용(실행·성공률)·오류(실패·오류율)·보안(이슈·critical)

6.1 ORB 심사 버그 수정

에이전트 게시 심사 보드: 심사요청(임시등록)→심사→승인 시 게시(워크플로우 lifecycle 연결). 자동채점(5영역)·재실행·자동/수동 배지 제공. 수정 임시등록 입력창이 한 글자만 입력되던 포커스 손실 버그 해결(연속 입력 정상).

9.1 Risk 워크스페이스 · Anomalies 재설계

Risk 워크스페이스: 에이전트 운영 리스크(보안·품질·이상·비용)를 카테고리/심각도/에이전트별로 집계, 알람 피드·상세·조치(완료/에스컬레이션/차단/무시). Anomalies: 실제 이상 탐지(지연추세·정확도드리프트·토큰급증·오류급증·보안패턴)를 목록·요약·히트맵으로 제공.

10. 외부 Agent 연동 — 수집 SDK 온램프 신규

METIS 밖에서 운영하는 에이전트(LangChain·자체 서비스 등)도 1줄 연동으로 동일한 4게이트 평가·거버넌스·성과 체계에 편입할 수 있습니다.

키 발급TENANT_ADMIN: POST /v1/ingest/keys → mts_live_…(1회 노출)
적재 APIPOST /v1/ingest/runs (Authorization: Bearer mts_live_…) 단건/배치, 멱등(runId)
Python SDKsdks/python — 무의존성·비차단
from metis import Metis
metis = Metis(api_key="mts_live_…", base_url="https://app.metis.ai")

@metis.eval(agent="sr-impact-analyzer", system="SR/ITSM")
def run_agent(question):
    return llm.invoke(question)   # 반환값 자동 파싱 → 비동기 적재·평가
적재된 외부 실행은 내부 워크플로우와 동일한 평가 엔진을 거쳐 Agent Execution·효과성·Anomalies 화면에 함께 표시됩니다. (검증: 정상/환각/프롬프트인젝션/시크릿유출 4종 시나리오 평가 통과)

11. 효과성 원천 수집 신규

MTTD·커버리지를 실측하려면 원천신호를 보냅니다. POST /v1/metrics/effectiveness-signal (DETECTION: 발생→탐지 시각 / COVERAGE: 테스트 결과). 신호가 없으면 MTTD는 근사, 커버리지는 목표값으로 표기됩니다.

12. 설치 · 실행

스키마 변경(효과성·시스템·코드·온램프 컬럼 등)이 누적되어 있으므로 최초/업데이트 시 db push·generate·seed가 필요합니다.
