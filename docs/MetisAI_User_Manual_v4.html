METIS.AI 사용자 매뉴얼

상세 활용 가이드 v4 — "무엇"이 아니라 "어떻게 쓰는가"를 단계별로 (2026-06 현행)

1. 개요 · 핵심 개념

METIS.AI는 AI 에이전트를 통제(거버넌스)하고 성과(공수 절감·ROI)를 증명하는 플랫폼입니다. 먼저 4개 핵심 개념을 이해하면 모든 화면이 쉬워집니다.

개념의미현실 비유
Main Agent (워크플로우)하나의 업무를 수행하는 에이전트. 예: "SR 영향도 분석 Agent"담당 직원 1명
Sub Agent (노드)워크플로우를 구성하는 단계. 예: 요구분석→영향산출그 직원이 하는 세부 작업들
4게이트 평가실행마다 자동 채점: ①정확도 ②환각 ③보안 ④정책품질검사 검문소
효과성(성과)수작업 대비 절감 공수·ROI·MTTD/MTTR"이 직원이 月 몇 MM을 아꼈나"
💡 모든 에이전트는 화면에서 코드 · 이름으로 표시됩니다(예: DEV-003 · Dev Agent). 코드는 분류용 식별자입니다.

2. 설치 · 로그인 · 권한

2.1 서버 실행 (관리자/개발자)

  1. Docker Desktop을 먼저 실행합니다(PostgreSQL·Redis 컨테이너에 필요).
  2. 프로젝트 폴더의 start-metis.bat를 더블클릭합니다. (자동으로 ①사전점검 ②환경변수 ③Docker ④의존성 ⑤DB push/seed ⑥빌드 ⑦서버 3개 기동)
  3. 브라우저에서 http://localhost:3000 접속 → 로그인 화면.
⚠ EPERM(파일 잠금) 오류가 나면: 떠 있는 Metis 창을 모두 닫고 → 새 명령창에서 taskkill /F /IM node.exe → bat 재실행. 폴더가 OneDrive 동기화 폴더면 동기화를 일시중지하거나 저장소를 C:\dev\ 같은 비동기 경로로 옮기세요.

2.2 로그인 & 첫 화면

  1. 이메일·비밀번호 입력 후 로그인 → 자동으로 홈 대시보드(/home)로 이동합니다.
  2. 세션은 자동 갱신됩니다. 15분 이상 작업해도 로그아웃되지 않으며, 리프레시 실패 시에만 로그인 화면으로 돌아갑니다.

2.3 권한(역할)별 할 수 있는 일

PLATFORM_ADMIN플랫폼 전체·테넌트 관리

3. 홈 대시보드 사용법

메뉴: 좌측 "대시보드" 또는 상단바 "대시보드" → /home

홈은 운영 현황을 한 화면에 보여줍니다. 위에서 아래로 ①요약 ②효과성 ③활용 ④추이 ⑤APM ⑥Agent 현황 ⑦런처 순입니다.

3.1 기간 선택 · 빠른실행

  1. 우측 상단 7/30/90/180일 버튼으로 집계 기간을 바꿉니다 → 모든 수치가 즉시 재계산됩니다.
  2. 그 왼쪽 빠른실행 버튼(최근 1개·인기 2개)을 누르면 자주 쓰는 에이전트를 바로 실행할 수 있습니다.
  3. ⟳(새로고침)으로 최신 데이터를 다시 불러옵니다.

3.2 요약 카드 6종 — 클릭하면 무엇이 나오나

카드의미클릭 시
Agent 총 수행기간 내 총 실행 건수전체 실행 이력 팝업
수행 성공률성공 비율성공 이력 팝업
평균 속도평균 응답 지연(ms)실행 이력
토큰 비용기간 토큰 비용($)비용 이력
이상 감지이상 탐지 건수(빨강 강조)실패/이상 이력
활용 시스템 신규활용 중 시스템 수시스템별 사용 Agent 팝업 — 어느 시스템에서 어떤 에이전트가 몇 회 쓰였는지
활용 예: "활용 시스템 6개"를 누르면 "SR/ITSM 시스템: OPS-004·EXT-002·EXT-003 (총 1,850회)"처럼 시스템 단위로 어떤 에이전트가 얼마나 쓰이는지 즉시 파악 → 시스템별 AI 도입 현황 보고에 활용.

3.3 효과성 요약 — 공수·ROI 읽는 법

  1. 총 절감 공수는 MM(맨먼스, 1MM=160시간) 단위입니다. 예 "3.2 MM" = 사람 3.2명이 한 달 할 일을 에이전트가 대체.
  2. 카드를 클릭하면 Agent별 절감 효과 팝업이 열립니다(에이전트·시스템·절감 공수 MM·절감률·순가치 $).
  3. ROI는 인건비 환산 대비 비용 배수입니다. 금액은 천단위로 표기됩니다.
  4. 품질 평균 증감 카드를 클릭하면 Agent별 품질 증감 팝업(이전 기간 대비 ▲▼)이 나옵니다.
📈 절감률 계산 원리: (수작업 분 − AI 분) ÷ 수작업 분. 예) SR 영향도 분석이 사람 30분 → AI 10분이면 66.6% 절감. 이 "수작업 분" 기준은 4.3에서 직접 입력합니다.

3.4 활용 랭킹 (Top3 / 미활용 Top3)

  1. 가장 많이/적게 쓰는 에이전트 3개씩 표시됩니다(0회 = 미사용, 노랑 강조).
  2. 에이전트 행을 클릭하면 사용 상세 팝업: 어느 시스템에서·몇 회·성공률·점수·절감 공수.
활용 예: 미활용 Top3에 "EXT-003 · SR 영향도(편의) 0회"가 보이면 → 도입했지만 안 쓰이는 에이전트 발견 → 폐기 또는 활성화 의사결정.

3.5 성과·품질 추이

  • 성과 추이: 막대=실행량(건), 선=성공률(%). 가로축=날짜, 세로축=0~100.
  • 품질 추이: 선=종합점수(0~100), 막대=이상(건).
  • "상세 ▾"=드릴다운 팝업, "분석 →"=성과/품질 전용 분석 페이지로 이동.

3.6 APM X-View — 느린/실패 수행 찾기 신규

제니퍼 모니터링 스타일. 가로축=하루 24시간(시각), 세로축=호출 소요시간(ms), 점=개별 수행(성공=초록/실패=빨강/실행중=노랑).

  1. 화면 위쪽(=느린 수행)이나 특정 시간대에 점이 몰린 곳을 찾습니다.
  2. 마우스로 그 영역을 드래그(사각형 선택)합니다.
  3. 선택된 수행들의 상세 팝업(건수·평균 소요·실패 수·주요 Agent·목록)이 열립니다 → 어떤 에이전트가 언제 느렸는지 원인 추적.
활용 예: "오후 2시경 상단(지연 2초+)에 빨간 점이 몰림"을 드래그 → "OPS-005 이벤트 대응 Agent가 8건 실패, 평균 3,100ms" 확인 → 해당 에이전트 점검.

3.7 Agent 현황 · 런처

  1. Agent 현황(Main Agent별): 상위 10개가 한눈에, 나머지는 스크롤. 막대=평균점수, 색=상태, 우측=품질추이·실행수·이상.
  2. 행을 클릭하면 상세 팝업: 효과성 블록(수작업→AI분·절감률·절감 공수 MM·순가치) + Sub Agent(노드)별 점수/이상/비용/지연/보안 표 + "워크플로우 빌더 열기".
  3. 하단 Agent 목록에서 "▶ 실행"으로 즉시 실행, 최근 사용에서 바로가기.

4. Agent 운영·관리

4.1 카테고리별 Agent 목록 · 실행 이력 상세

메뉴: 운영 Agent / 개발 Agent / 품질·공통 Agent / 편의 Agent

  1. 카테고리 메뉴를 열면 해당 에이전트 카드 목록이 보입니다.
  2. 에이전트를 선택하면 실행 이력이 열리고, 각 실행을 펼치면 7개 측면을 확인합니다: 품질·보안·비용·오류사항·관련 지식·수행시간·정책.
  3. 품질 탭: 종합점수·정확도·환각률·등급(A~F). 보안 탭: 위험도·입력위협·출력유출. 오류 탭: 발생 오류와 자동 수집된 패턴.

4.2 Agent 실행 테스트

메뉴: AGENT > Agent 실행 테스트

  1. 실행할 에이전트(워크플로우)를 선택합니다.
  2. 입력값(질문/대상)을 넣고 "실행"을 누릅니다.
  3. 실행이 끝나면 결과와 함께 4게이트 평가 결과(점수·보안위험·이상)가 자동으로 표시됩니다. 별도 평가 호출이 필요 없습니다(엔진이 자동 채점).

4.3 ⭐ Agent 기준/시스템 편집 (효과성의 출발점)

메뉴: 상단바 "Agent 관리/통제" > 📐 Agent 기준/시스템 탭 · 권한: TENANT_ADMIN/OPERATOR

효과성(절감 공수·ROI)이 계산되려면 "사람이 직접 하면 몇 분 걸리는지" 기준을 먼저 입력해야 합니다. 여기가 그 입력 화면입니다.

  1. "Agent 관리/통제" 화면 상단 탭에서 📐 Agent 기준/시스템을 선택합니다.
  2. 에이전트별 행에서 다음을 입력/수정합니다:
    • 시스템명: 이 에이전트가 쓰이는 시스템(예: "SR/ITSM 시스템") — 시스템별 분석의 기준이 됩니다.
    • 수작업 분/회: 사람이 직접 하면 걸리는 시간(예: 30) — 절감률 계산 기준.
    • 가치 라벨(예: "영향도 분석 시간 절감"), 도메인, MTTD 목표(%), 커버리지 목표(x).
  3. 행별 "저장" 또는 상단 "전체 저장"을 누릅니다. 저장 성공 배너가 뜨면 반영 완료.
  4. 이후 실행부터 홈 효과성·성과/효과성 화면에 절감 공수·절감률·ROI가 이 기준으로 계산되어 나타납니다.
활용 예: "SR 영향도 분석 Agent" 시스템명="SR/ITSM 시스템", 수작업 분/회=30 입력 → 에이전트 실제 평균 10분 → 자동으로 66.6% 절감·MM·순가치 계산되어 대시보드에 표시.
⚠ "수작업 분/회"를 입력하지 않으면 절감이 0으로 계산됩니다. 효과성을 보려면 반드시 먼저 설정하세요.

5. 거버넌스

5.1 정책코드 엔진 — 정책 만들기

메뉴: 거버넌스 > 정책코드 엔진

  1. "정책 생성"을 누릅니다.
  2. 정책 키·설명·조건(예: 보안 위험도 high 이상 차단, 종합점수 70 미만 경고)을 입력합니다.
  3. "저장"하면 정책이 활성화되어, 이후 모든 평가가 이 정책을 함께 검사합니다(위반 시 알람 발생).

5.2 평가 Gate 설정 — 4게이트 임계값

메뉴: 거버넌스 > 평가 Gate 설정

  1. 정확도·환각·보안·정책 게이트별 통과 임계값을 조정합니다(예: 종합점수 하드게이트 70).
  2. LLM 심사 샘플링 비율을 설정합니다(전수 평가는 비용↑ → 일부만 LLM 심사).
  3. 저장하면 즉시 평가 기준에 반영됩니다.

5.3 정책 제안 — 자동 제안 검토

  1. 시스템이 실행 패턴을 분석해 정책을 자동 제안합니다.
  2. "정책 제안" 화면에서 제안을 검토하고 승인/거절합니다.
  3. 승인하면 정책코드 엔진에 등록됩니다.

5.4 Evidence Pack — 규제 증거 묶음

감사 로그·평가 기록·정책 적용 내역을 묶어 규제 대응(AI 기본법 등) 증거로 내보냅니다. 화면에서 기간·대상을 선택해 생성합니다.

5.5 ORB 심사 — 에이전트 게시 승인 절차

메뉴: 거버넌스 > ORB 심사

새 에이전트를 운영에 정식 게시하기 전 심사하는 보드입니다.

  1. 심사요청(임시등록)을 누르고 폼에 Agent Key·이름·버전·요청자·팀을 입력합니다. (입력은 연속으로 정상 동작합니다.)
  2. 제출하면 심사 대기 목록에 올라가고, 자동채점(5영역) 결과가 표시됩니다.
  3. 심사위원이 5영역을 검토(자동/수동 배지로 구분)하고 필요 시 "재채점"을 실행합니다.
  4. 승인하면 해당 워크플로우가 게시(lifecycle)되어 운영에 노출됩니다.

5.6 AI 활동 로그 · KPI 대시보드

AI 활동 로그=감사 추적(누가·무엇을·언제). KPI 대시보드=실행 메트릭·성공률·정책 위반·ROI 요약.

6. 성과/효과성 분석 — 의사결정용

메뉴: 거버넌스 > 성과/효과성

요약 / 에이전트별 / 시스템별 3단으로, "AI가 만든 가치"와 "어디가 문제인지"를 동시에 봅니다.

MTTR평균 복구 시간(FDS 알람 생성→해결 실측).
커버리지테스트 커버리지(실측 또는 목표).
시스템별 롤업시스템 단위 활용(실행·성공률)·오류(실패·오류율)·보안(이슈·critical) → "어느 시스템이 오류·보안이 많은가"로 개선 우선순위 결정.
  1. 기간(7/30/90)을 선택합니다.
  2. 요약 카드로 전체 절감·ROI·평균 MTTR을 확인합니다.
  3. 에이전트별 표에서 절감률 낮은/오류 많은 에이전트를 식별합니다.
  4. 시스템별 롤업에서 오류율·보안 이슈가 높은 시스템을 찾아 개선 대상으로 정합니다.

7. 지식관리 — 오류를 자산으로

메뉴: 지식관리 (Registry / Artifact / 오류 패턴 / Pipeline)

자동 학습 루프

실행 중 오류 발생 → 오류 패턴 자동 수집 → 지식으로 정리 → 반복 시 정책으로 승격 → 다음 실행 때 그 지식이 주입되어 같은 실수 방지.

  1. 오류 패턴 관리: 자동 수집된 오류 패턴(발생 횟수·심각도)을 확인합니다.
  2. 반복되는 패턴을 지식(Artifact)으로 등록하거나 정책 승격 후보로 표시합니다.
  3. Knowledge Pipeline에서 지식이 얼마나 활용되는지(활용/미활용)를 보고, 미활용 지식은 정리합니다.

8. 오케스트레이션 — 워크플로우 만들기

메뉴: 오케스트레이션 (워크플로우 빌더 / 템플릿 마켓 / 실행 모니터링 / 커넥터 관리)

8.1 워크플로우 빌더로 에이전트 만들기

  1. 워크플로우 빌더를 열고 새 워크플로우를 생성합니다(또는 템플릿 마켓에서 시작).
  2. 캔버스에 노드(sub agent)를 추가합니다(데이터 수집·AI 처리·조건·알림·커넥터 등).
  3. 노드를 연결해 실행 순서를 만들고, 각 노드의 설정(모델·프롬프트·커넥터)을 지정합니다.
  4. 저장 후 실행하면 ExecutionSession이 생성되고 각 노드가 자동 평가됩니다.

8.2 템플릿 마켓 · 실행 모니터링 · 커넥터 관리

  • 템플릿 마켓: 미리 만든 스타터 워크플로우를 가져와 빠르게 시작.
  • 실행 모니터링: 진행/완료/실패 현황과 이력.
  • 커넥터 관리: 외부 시스템 연결(노드)과 거버넌스(서킷·SSRF 차단 등).

9. 인사이트 · FinOps

9.1 FinOps · FinOps 데모 — 비용 절감 확인

메뉴: 인사이트 > FinOps / FinOps 데모

  1. FinOps 데모에서 프롬프트를 입력하고 "최적화 실행"을 누릅니다.
  2. Gate1 캐시: 의미가 비슷한 이전 요청이 있으면 HIT(유사도 % 표시) → 비용 100% 절감. 예) "테트리스 개발해줘" 후 "테트리스 게임 개발해줘"도 의미 유사도로 HIT.
  3. Gate2 라우팅: 난이도에 맞는 저비용 모델로 라우팅. Gate3 패커: 프롬프트 압축. 결과로 절감액·캐시 적중률을 확인합니다.

9.2 Agent Evaluator

4게이트 품질 엔진의 결과 모음: 등급 분포·환각률·에이전트별 품질. 품질 저하 에이전트를 찾는 화면.

9.3 Anomalies (이상 감지)

  1. 기간·심각도·유형(지연추세/정확도드리프트/토큰급증/오류급증/보안패턴)으로 필터합니다.
  2. 요약·히트맵·목록으로 언제·어떤 이상이 났는지 파악합니다.

9.4 Risk 워크스페이스 — 알람 조치

메뉴: 인사이트 > Risk 워크스페이스 · 권한: OPERATOR

  1. 카테고리(보안/품질/이상/비용)·심각도·에이전트별로 리스크 알람을 봅니다.
  2. 알람을 선택해 상세(연관 알람 포함)를 확인합니다.
  3. 조치완료 / 에스컬레이션 / 차단 / 무시 중 하나를 실행합니다 → 상태가 갱신되고 MTTR(복구 시간)에 반영됩니다.

10. 외부 Agent 연동 — 수집 SDK 신규

대상: METIS 밖(LangChain·자체 서비스 등)에서 이미 운영 중인 에이전트

METIS에서 만들지 않은 외부 에이전트도 평가·거버넌스·성과 체계에 편입할 수 있습니다.

  1. 키 발급(TENANT_ADMIN): POST /v1/ingest/keys 또는 설정 화면에서 발급 → mts_live_…(1회만 표시되니 복사).
  2. 설치: pip install로 SDK 설치(저장소 sdks/python).
  3. 연동: 에이전트 함수에 데코레이터 한 줄을 붙입니다.
from metis import Metis
metis = Metis(api_key="mts_live_…", base_url="https://app.metis.ai")

@metis.eval(agent="sr-impact-analyzer", system="SR/ITSM 시스템")
def run_agent(question):
    return llm.invoke(question)   # 반환값 자동 파싱 → 비동기 적재·평가
  1. 실행 후 홈 Agent Execution·효과성·Anomalies에 외부 에이전트가 내부와 동일하게 평가되어 나타납니다.
검증 완료: 정상/환각/프롬프트인젝션/시크릿유출 4종 시나리오가 동일 엔진으로 평가됨(품질·보안·이상 차등 채점).

11. 효과성 원천 수집 — MTTD/커버리지 실측

MTTD·커버리지를 "목표"가 아닌 "실측"으로 채우려면 원천신호를 보냅니다.

# 탐지 시간(MTTD): 문제 발생→탐지 시각
POST /v1/metrics/effectiveness-signal
{ "workflowKey":"ops-service-monitoring", "kind":"DETECTION",
  "occurredAt":"2026-06-04T01:00:00Z", "detectedAt":"2026-06-04T01:03:30Z" }

# 커버리지: 테스트 결과
{ "workflowKey":"dev-test-agent", "kind":"COVERAGE",
  "testsTotal":200, "testsPassed":188, "coveragePct":86 }

신호가 쌓이면 성과/효과성 화면의 MTTD·커버리지가 "실측"으로 전환됩니다(없으면 근사/목표 표기).

12. 실전 End-to-End 시나리오

시나리오 A — 신규 운영 에이전트 도입부터 성과 증명까지

  1. 만들기: 워크플로우 빌더(8장)로 "SR 영향도 분석" 워크플로우를 노드로 설계·저장.
  2. 심사: ORB 심사(5.5)에서 임시등록→자동채점→승인→게시.
  3. 기준 설정: Agent 기준/시스템(4.3)에서 시스템명="SR/ITSM", 수작업 분/회=30 입력.
  4. 실행: Agent 실행 테스트(4.2) 또는 운영에서 실제 수행 → 4게이트 자동 평가.
  5. 성과 확인: 홈 효과성(3.3)·성과/효과성(6장)에서 절감 공수(MM)·ROI·품질 확인.
  6. 리스크 관리: 이상/위반 발생 시 Risk 워크스페이스(9.4)에서 조치 → MTTR 개선.
  7. 거버넌스 보고: Evidence Pack(5.4)으로 규제 증거 내보내기.

시나리오 B — 기존 외부(LangChain) 에이전트를 METIS로

  1. 수집 키 발급(10장) → SDK 설치 → @metis.eval 1줄 적용.
  2. 기준/시스템(4.3)에서 해당 에이전트의 수작업 분·시스템명 입력.
  3. 홈·효과성에서 외부 에이전트의 품질·절감·이상이 내부와 함께 집계되는지 확인.

시나리오 C — 비용 급증 대응

  1. 홈 "토큰 비용" 급증 확인 → APM X-View(3.6)로 비용·지연 큰 수행 구간 드래그 선택.
  2. FinOps(9.1)에서 캐시 적중률·라우팅 확인, 평가 Gate(5.2)에서 LLM 심사 샘플링 조정.
  3. 성과/효과성 시스템별 롤업으로 비용 집중 시스템 식별.
