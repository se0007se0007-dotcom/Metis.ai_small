1플랫폼 개요

Metis.AI는 조직이 보유한 다양한 AI 에이전트(사내 워크플로우형 + 외부 전용화면형)를 하나의 통제면(control plane)에서 등록·실행·평가·감사하도록 만드는 플랫폼입니다. 모든 실행은 4‑게이트 평가와 감사 로그를 거쳐 대시보드에 집계됩니다.

2주요 에이전트 형태 (사내 파이프라인 / 외부 전용화면)
4게이트: 품질 · 비용 · 보안 · 이상
6RBAC 역할
80+도메인 데이터 모델(Prisma)

운영 통제면

실행 이력·비용·품질·보안·이상을 테넌트 단위로 집계하고, 운영 기준값으로 등급/헬스/ROI를 산출.

거버넌스

정책(Policy)·ORB 심사·FDS 이상탐지·자동조치(AutoAction)·감사로그(AuditLog)로 에이전트 행위를 통제.

기준정보(Master Data)

Agent·모델단가·사용자·테넌트/팀·운영기준값을 관리자 화면에서 직접 편집, 모든 화면이 이를 참조.

3시스템 아키텍처

브라우저 → Next.js(웹) → NestJS(API) → PostgreSQL/Redis. 외부 전용화면 에이전트는 iframe으로 임베드되어 실행 결과를 API로 보고하고, FinOps 플랫폼이 LLM 호출·원장(비용)을 담당합니다.

웹 ↔ API

Next.js가 /v1 REST를 호출(TanStack Query 캐시). 인증은 JWT 쿠키/베어러.

외부 전용화면

에이전트 고유 화면을 iframe으로 임베드 → 실행 후 postMessage → external-record가 ingest+감사 기록.

FinOps

LLM 호출은 gateway 경유, 토큰·비용은 control‑plane 원장에 적재. test‑agent는 예시 외부 에이전트.

4논리 아키텍처 (도메인)

핵심 도메인은 6개 묶음으로 나뉩니다. 모든 도메인은 Tenant를 루트로 격리됩니다.

핵심 개념: "에이전트"의 표현

본 플랫폼에는 별도의 Agent 엔티티가 없습니다. 메인 에이전트 = Workflow, 서브 에이전트 = WorkflowNodeDef로 표현하며, 에이전트별 설정(전용 실행화면 launchUrl, 대상 시스템 system, 수작업 분 manualMinutesPerRun, 시급 hourlyRateUsd)은 Workflow.effectivenessJson에 저장합니다. 게시 여부는 listed(ORB 승인 시 카탈로그 노출)로 통제합니다.

7핵심 실행 흐름

7‑1. 사내 워크플로우 에이전트 — 4‑게이트 평가

저장된 에이전트를 key로 즉시 실행하면 PipelineEngine.execute()가 노드 실행기(NodeExecutorRegistry)로 각 노드를 수행하고, 노드마다 EvaluatorService.evaluate()가 4‑게이트를 평가해 AgentEvaluation으로 적재합니다.

7‑2. 외부 전용화면 에이전트 — 임베드 + 결과 보고

전용 실행화면 launchUrl이 설정된 에이전트는 metis 안에 iframe으로 임베드됩니다. 사용자가 그 화면에서 실행하면 화면이 postMessage로 결과(출력·비용·지연·구간 타이밍)를 부모(metis)로 넘기고, metis는 /workflows/external-record → IngestService로 ExecutionSession + 4‑게이트 + 구간(Step) + 감사로그를 적재합니다.

7‑3. SDK Ingest (서버‑투‑서버)

외부 시스템은 Authorization: Bearer mts_… (또는 x-metis-key)로 /v1/ingest에 실행 결과를 보고할 수 있습니다. 동일 EvaluatorService를 거쳐 사내 실행과 같은 4‑게이트 평가·적재가 이뤄집니다. 키 scope로 agentName을 제한해 스푸핑을 방지합니다.

일관성 포인트: 사내 파이프라인 · 외부 전용화면 · SDK 적재 — 세 경로 모두 동일한 EvaluatorService를 통과합니다. 즉 어디서 실행되든 같은 4‑게이트 기준으로 평가·기록됩니다.

8기능 구성도 (사용자 관점)

사용자가 실제로 마주하는 메뉴·기능을 업무 흐름 순서로 배치한 구성도입니다.

9보안 · 거버넌스

인증·인가

JWT + Passport, bcrypt 자격증명. RBAC 6역할로 메뉴/엔드포인트 권한 분리(예: 기준정보 편집은 TENANT_ADMIN/PLATFORM_ADMIN).

멀티테넌시 격리

모든 도메인 쿼리에 tenantId 강제(withTenantIsolation). SDK 적재 키는 scope로 agentName 허용목록 제한.

감사·추적

@Audit 인터셉터가 모든 변경/실행을 AuditLog로 기록. correlationId로 실행‑평가‑감사 연결.

입력 안전

글로벌 ValidationPipe(화이트리스트), ingest 단계에서 비밀정보 redaction, 출력 유출 카운트(보안 게이트).

거버넌스 통제

Policy(비용·인증·감사 규칙) · ORB 심사(게시 승인) · FDS 이상탐지 → AutoAction(차단·격리·승인요구) · GovernanceDecision.

비용 가버넌스

모델단가 기준정보 + FinOps 원장(토큰/비용)으로 실행별 비용을 산출·집계, 운영기준값으로 등급화.

운영 주의: .env에는 실제 LLM 키가 포함되며 절대 커밋·노출 금지(gitignore). 시뮬레이터는 SIM_USE_REAL_API=0으로 모킹 유지 가능.

기동 (start-metis.bat, 7단계)

  • 전제확인 → 환경변수 → FinOps 컨테이너 → PostgreSQL/Redis → DB 셋업(generate·push) → 공유패키지 빌드 → 앱 기동(turbo)
  • 시드는 최초 1회만 실행됩니다(.metis_seeded 마커). 이후 재시작은 시드를 건너뛰어 사용자 편집(launchUrl)·실행 이력이 보존됩니다.
  • 데모 데이터를 새로 깔려면 .metis_seeded를 삭제 후 재기동.
시드(seed-dashboard.ts)는 사용자 데이터를 덮어쓰지 않도록 보강되었습니다: 워크플로우 effectivenessJson은 사용자 값 우선 병합, 실행 이력은 시드가 만든 데모(inputJson.seeded=true)만 정리하고 실제 실행은 보존.

관측·확장

Prometheus/Grafana로 메트릭 수집, BullMQ 워커로 장시간 작업 수평 확장. 릴리즈 엔지니어링(Replay/Shadow/Canary/Promotion)으로 버전 승격을 통제.
