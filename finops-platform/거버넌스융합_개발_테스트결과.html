2. 핵심: "거버넌스·품질·비용 동시 만족 폐루프"

기존의 품질게이트 강등(승인/카나리/보류) 위에 Metis의 거버넌스 3요소를 결합했습니다.

에이전트 호출 (X-Metis-Data-Class / Risk-Score / Policy-Hash) │ ▼ ① 캐시 정책 판정 (Patent 3) ├─ PII·기밀 등급 → 캐시 재사용 차단 (DENY_SENSITIVE_DATA) ├─ 고위험(≥0.7) → 캐시 재사용 차단 (DENY_HIGH_RISK) └─ 정책해시 스코프 캐시키 → 정책 변경 시 자동 무효화 │ ▼ ② 라우팅 결정 ├─ 고위험(≥0.8) → 예산 강등 '방어' + 안전 최소 티어로 상향(escalate) ├─ 그 외 + 예산 압박 → 품질게이트 통과 후에만 강등(downgrade) └─ 비용·품질·거버넌스를 한 번에 만족시켜 라우팅 사전 결정 │ ▼ ③ 원장 기록 (cache_decision / gov_action / data_class / risk_score) ▼ ④ 거버넌스 뷰: 차단 건수 · 강등 방어 · 민감 캐시 누출 0 · 준수율
이 결합은 단순 비용 라우터(CloudZero·Vantage·오픈소스 게이트웨이)가 흉내 낼 수 없는 해자이며, 특허 청구 대상입니다. "비용을 아끼다 민감정보가 캐시로 새거나, 예산 압박에 고위험 작업이 약한 모델로 떨어지는" 사고를 원천 차단합니다.

6. 직접 확인 방법

stop.bat → run.bat 재실행 후 대시보드(:8500) 왼쪽 🛡️ 거버넌스 메뉴. 시뮬레이터가 CRM사업팀 상담봇의 30% PII 문의·코드리뷰 35% 고위험을 발생시켜, 캐시 차단·리스크 상향·준수율 100%가 실시간으로 채워집니다. 단위테스트는 python tests\governance_unit.py로 즉시 실행 가능합니다.

테스트 환경 안내: 본 개발 환경의 샌드박스 파일 동기화 일시 오류로 전체 tests/e2e.py(25 시나리오)는 샌드박스에서 직접 구동하지 못해, 거버넌스 핵심을 단위(16)+통합(10) 테스트로 분리 검증했습니다(총 26/26 통과). 사장님 PC의 실제 소스는 정상이며, run.bat+tests/e2e.py로 전체 통합 테스트를 로컬에서 확인하실 수 있습니다.
