1. 개요 · 아키텍처 원칙

METIS.AI는 멀티테넌트 AgentOps 거버넌스·성과(ROI) 플랫폼입니다. Turborepo 모노레포(3 apps / 4 packages), 백엔드 NestJS, 프론트 Next.js 15, 영속 PostgreSQL(Prisma), 큐/캐시 Redis(BullMQ)로 구성됩니다.

원칙구현
도메인 모듈화NestJS 27개 도메인 모듈(평가/거버넌스/FinOps/지식/수집 등) 분리
단일 평가 엔진내부 워크플로우·외부 SDK 모두 동일한 EvaluatorService.evaluate()를 통과
멀티테넌시모든 도메인 데이터에 tenantId 스탬프 + 서버측 강제 격리
보안 기본값httpOnly 쿠키+자동갱신, CSRF, RBAC(6역할), SSRF/프롬프트가드, CSP nonce
관측·복원력OpenTelemetry 계측, best-effort 훅(평가 실패가 실행을 막지 않음), 큐 재시도

5. 논리 아키텍처 — 핵심 흐름

5.1 평가 파이프라인 (모든 실행의 심장)

워크플로우 실행 시 PipelineEngine이 노드를 순회하며, 각 노드 실행 직후 동일한 EvaluatorService.evaluate()를 호출합니다(best-effort, 실패해도 파이프라인 미중단).

PipelineEngine.execute()→노드 loop→executor.execute()→ExecutionStep 저장→evaluate()
EvaluatorService.evaluate() 내부
Gate1·2 품질정확도·환각·점수 (quality-evaluator)
LLM 심사샘플링(adaptive) · llm-judge
Gate 보안프롬프트 인젝션·출력유출·툴체인 (security-evaluator + prompt-guard)
▼
Gate 비용/정책cost-evaluator + 활성 정책
이상 탐지z-score·IQR·회귀·에러서지 (anomaly-detector, DB 히스토리)
종합점수·저장computeOverallScore → AgentEvaluation
▼
알람정책 위반 → FDSAlert(raiseAlarms)
지식 캡처오류 → ErrorPattern/KnowledgeArtifact

그림 5-1. 4게이트+이상+알람+지식의 단일 평가 파이프라인

5.2 수집 온램프 (외부 Agent → 동일 엔진)

외부 Agent + @metis.eval→POST /v1/ingest/runs (API키)→IngestKeyGuard(tenant 해석)→redactSecrets·멱등 upsert→ExecutionSession(source=sdk)→EvaluatorService.evaluate()→동일 대시보드

핵심: 평가 로직을 재구현하지 않고 "입구"만 추가 — 외부/내부 에이전트가 같은 화면에 동일 기준으로 집계.

5.3 FinOps 3게이트 (비용 최적화)

프롬프트→Gate1 의미 캐시(임베딩 코사인≥0.93 → HIT)→Gate2 모델 라우팅(난이도→tier)→Gate3 스킬 패커(압축)→FinOpsTokenLog

의미 캐시는 OpenAI 임베딩(text-embedding-3-small)로 코사인 유사도 매칭(키 없거나 차단 시 정확일치 폴백).

5.4 효과성(성과) 계산

Agent 기준(effectivenessJson: 수작업분·system)+실측(ExecutionSession latency·success)→절감공수=성공×수작업분 − 실제→MM·ROI·절감%

MTTD=원천신호(EffectivenessSignal DETECTION) 또는 평균지연 근사 · MTTR=FDSAlert createdAt→resolvedAt · 커버리지=COVERAGE 신호 · 시스템별 롤업(활용/오류/보안).

5.5 거버넌스 · ORB lifecycle

정책코드 엔진(Policy)→평가 시 정책 검사→위반 FDSAlert→Risk 워크스페이스 조치→감사로그·Evidence Pack
ORB 임시등록→자동채점(5영역)→심사→승인 → Workflow 게시(WorkflowVersion·VersionPromotion)

10. 시퀀스 다이어그램

10.1 내부 워크플로우 실행 + 평가

Web→POST /v1/workflow-nodes/execute-sync→PipelineEngine→executor.execute→ExecutionStep.create→evaluate()→AgentEvaluation/FDSAlert→결과 반환

10.2 외부 SDK 적재 + 평가

@metis.eval→/v1/ingest/runs→IngestKeyGuard→IngestService→ExecutionSession(sdk)→evaluate()→202/200

10.3 ORB 게시

임시등록→OrbAutoScore(5영역)→심사→승인→Workflow 게시(Version/Promotion)

10.4 FinOps 캐시 HIT

optimize(prompt)→embed→코사인≥0.93?→HIT: 캐시응답·100%절감|MISS: 라우팅·패커·로그(embedding 저장)

10.5 401 자동 세션 갱신

API 401→silentRefresh(/auth/refresh, single-flight)→성공: 원요청 재시도|실패: /login
