🎯 1. Metis.AI 란?

Metis.AI는 기업에서 AI를 안전하고 효율적으로 운영하기 위한 통합 관리 플랫폼입니다. 복잡한 코딩 없이도 자연어(한국어)로 자동화 워크플로우를 만들 수 있고, 보안 점검부터 보고서 생성까지 한 번에 처리할 수 있습니다.

이런 분들에게 유용합니다

💻
IT 운영팀
반복 업무를 자동화하고, AI 에이전트의 활동을 모니터링할 수 있습니다.
🔭
보안팀
소스코드 보안 점검, 모의해킹, 취약점 보고서를 자동으로 생성합니다.
💰
관리자 / 경영진
AI 비용을 투명하게 관리하고, 감사 로그와 정책 준수 현황을 확인합니다.
🛠
개발팀
워크플로우 템플릿을 만들어 팀에 공유하고, 안전한 배포 프로세스를 실행합니다.

주요 기능 한눈에 보기

기능 설명 누가 쓰나요?
워크플로우 빌더 한국어로 "아침 9시에 보안 점검해줘"라고 입력하면 자동으로 워크플로우 생성 모든 사용자
보안 점검 (Pentest) 소스코드 업로드 → AI가 8개 공격 벡터로 취약점 분석 보안팀, 개발팀
보고서 생성 점검 결과를 전문 HTML/Word 보고서로 자동 작성 모든 사용자
템플릿 마켓 만들어진 워크플로우를 저장하고 팀원과 공유 모든 사용자
FinOps AI 사용 비용 분석, 최적화 추천 관리자
거버넌스 감사 로그, 정책 관리, 증적 수집 감사자, 관리자
릴리스 관리 Canary 배포, Shadow 테스트로 안전한 업데이트 개발팀, 운영팀

🌊 5. 워크플로우 빌더 (Metis.flo)

Metis.AI의 핵심 기능입니다. 자연어(한국어)로 원하는 작업을 설명하면 AI가 자동으로 워크플로우를 만들어줍니다. 코딩이 필요 없습니다.

빌더 화면 접속

사이드바에서 오케스트레이션 > 워크플로우 빌더를 클릭합니다.

빌더 화면 구성 (3-Column 레이아웃)

[ 워크플로우 빌더 화면 ]
왼쪽 패널
💬 프롬프트 입력
⚡ 빠른 시작 템플릿
🔎 Planner 결과
중앙 캔버스
✎ 워크플로우 제목
🔗 노드 파이프라인
(⏰→🔍→🤖→📧)
오른쪽 패널

워크플로우 만들기 (Step by Step)

왼쪽 패널에 원하는 작업을 한국어로 입력

예시: "아침 9시에 호랑이 관련 기사들을 검색해서 요약하여 내 이메일로 발송"
예시: "GitHub에서 소스코드를 가져와서 보안 점검하고 보고서를 만들어줘"

"✨ 워크플로우 생성" 버튼 클릭

AI가 프롬프트를 분석하여 중앙 캔버스에 노드(작업 단위)들을 자동으로 배치합니다. 단축키: Ctrl + Enter

생성된 노드 확인 및 수정

중앙 캔버스에 나타난 노드를 클릭하면 오른쪽 패널에 상세 설정이 나타납니다. 필요에 따라 설정을 수정하세요.

워크플로우 제목 수정 (선택)

중앙 캔버스 상단의 제목을 클릭하면 편집 모드로 전환됩니다. 원하는 이름을 입력하고 Enter를 누르세요.

사용 가능한 노드 종류 (13종)

AI가 프롬프트를 분석해 적절한 노드를 자동으로 선택하지만, 수동으로 추가할 수도 있습니다:

아이콘 노드 이름 기능 설명
⏰ Schedule Trigger 워크플로우 시작 시점 설정 (즉시 실행, 예약 시간, 반복)
🔍 Web Search Google/Bing에서 키워드로 정보 검색
🤖 AI Processing Claude, GPT 등 AI 모델로 텍스트 분석/요약/생성
📧 Email Send 결과를 이메일로 발송
💬 Slack Message Slack 채널에 메시지 전송
💾 Data Storage 결과를 데이터베이스에 저장 / 감사 로그 기록
🌐 API Call 외부 REST API 호출
🔄 Data Transform 데이터 형식 변환 (JSON, CSV 등)
⚡ Condition/Filter 조건에 따라 다음 단계 분기
⏳ Wait/Approval 사용자 승인 대기 또는 시간 대기
📁 File Operation 파일 업로드, 읽기, 변환
🔬 Pentest Scanner 소스코드 보안 취약점 모의해킹 분석
📊 Log/Monitor 실행 로그 기록 및 모니터링

노드 설정 변경하기

캔버스에서 수정할 노드를 클릭

선택된 노드는 파란색 테두리로 강조됩니다.

오른쪽 설정 패널에서 값 변경

각 노드 타입마다 설정할 수 있는 항목이 다릅니다. 예를 들어 AI Processing 노드에서는 AI 모델, 프롬프트 템플릿, 온도(temperature) 등을 설정합니다.

🤖
AI Processing 노드 주요 설정
• 모델: claude-sonnet-4.6 (기본값) — 다른 모델로 변경 가능
• 프롬프트 템플릿: AI에게 전달할 지시사항
• FinOps 옵션: 캐시/라우터/패커 활성화로 비용 절감

빠른 시작 템플릿

프롬프트를 입력하기 어려울 때는 왼쪽 패널 하단의 빠른 시작 템플릿에서 미리 만들어진 워크플로우를 선택할 수 있습니다:

📰 뉴스 모니터링 + 보고
키워드 검색 → AI 요약 → 이메일 발송
🔬 보안 점검 파이프라인
파일 업로드 → 보안 스캔 → 보고서 생성
📊 정기 보고서 자동화

🔔 조건부 알림

▶ 6. 워크플로우 실행하기

실행 방법

워크플로우가 준비되었는지 확인

우측 상단의 Readiness 점수 배지를 확인합니다. 초록색(🟢)이면 바로 실행 가능합니다.

우측 상단 "▶ 실행" 버튼 클릭

실행이 시작되면 버튼이 "⏳ 실행 중..."으로 변경됩니다.

진행 상황 확인

캔버스 상단에 진행률 바가 나타나고, 각 노드에 상태 아이콘이 표시됩니다:

○ 대기중 ⚙️ 실행중 ✅ 완료 ❌ 실패

결과 확인

모든 노드가 완료되면 결과가 오른쪽 설정 패널에 표시됩니다. 보고서 생성 노드가 있다면 다운로드 링크가 나타납니다.

예약 실행

Schedule 노드에서 실행 시간을 설정하면 예약 실행이 됩니다. 예약된 워크플로우는 상단에 ⏰ 예약됨 배지로 표시되며, "예약 취소" 버튼으로 취소할 수 있습니다.

⚠️
실행 중 에러가 발생하면?
실패한 노드에 ❌ 표시가 나타나고, 해당 노드를 클릭하면 오른쪽 패널에 에러 상세 내용이 표시됩니다. 설정을 수정한 후 다시 실행할 수 있습니다.

📄 7. 보고서 확인 및 다운로드

보안 점검이나 AI 분석 워크플로우를 실행하면 전문 보고서가 자동으로 생성됩니다.

보고서 형식

형식 파일 확장자 특징
HTML .html 웹 브라우저에서 바로 열어 확인. 차트, 그래프 포함. 가장 보기 편함
Word .doc Microsoft Word로 열기 가능. 편집하여 재사용 가능

보고서 다운로드 방법

워크플로우 실행 완료 후 "문서 생성" 노드 클릭

오른쪽 패널에 보고서 미리보기와 다운로드 버튼이 표시됩니다.

원하는 형식의 다운로드 버튼 클릭

"HTML 다운로드" 또는 "Word 다운로드" 버튼을 클릭하면 파일이 자동 다운로드됩니다.

보고서에 포함되는 내용 (보안 점검 예시)

📊 요약 대시보드

각 취약점의 이름, 위험도, 위치, 설명, 위험성, 수정 방안 포함
✅ 수정 권고
각 발견 항목별 구체적인 수정 코드와 조치 방법
⚠️ 위험성 분석
취약점이 악용될 경우 발생 가능한 구체적 피해 설명

• 퍼블리시(📦): 버전 스냅샷을 만들어 팀에 공식 배포합니다.

📜 11. 거버넌스 (감사/정책)

AI 활동의 투명성과 규정 준수를 보장하는 기능들입니다. 주로 감사자와 관리자가 사용합니다.

11-1. 감사 로그 (AI 활동 로그)

사이드바 거버넌스 > AI 활동 로그에서 확인합니다.

플랫폼에서 발생하는 모든 작업이 자동으로 기록됩니다: 워크플로우 생성/수정/삭제, 실행, 정책 평가, 팩 설치 등 누가 언제 무엇을 했는지 추적할 수 있습니다.

필드 설명
시간 작업이 발생한 정확한 시각
액션 CREATE, UPDATE, DELETE, EXECUTE, PUBLISH 등
대상 어떤 리소스에 대한 작업인지 (워크플로우, 팩, 정책 등)
사용자 작업을 수행한 사람의 ID
Correlation ID 하나의 요청을 처음부터 끝까지 추적하는 고유 ID

11-2. 정책 엔진

사이드바 거버넌스 > 정책코드 엔진에서 관리합니다.

거버넌스 정책을 정의하고, 워크플로우 실행 시 자동으로 정책 준수 여부를 평가합니다.

"정책 생성" 버튼 클릭

화면 우측 상단의 버튼을 클릭합니다.

정책 정보 입력

정책 이름, 유형(보안/규정준수/운영), 활성 여부, 규칙을 입력합니다.

저장 후 자동 적용

활성 상태의 정책은 워크플로우 실행 시 자동으로 평가됩니다.

11-3. Evidence Pack (증적 팩)

감사 대응을 위한 증적 자료를 자동으로 수집합니다. 월별, 작업 유형별로 정리되며 내보내기가 가능합니다.

11-4. KPI 대시보드

총 실행 수, 성공률, 평균 지연시간, 정책 위반 건수 등 핵심 운영 지표를 한눈에 확인합니다.

💰 12. FinOps 비용 관리

사이드바 인사이트 > FinOps에서 접속합니다. AI API 호출 비용을 분석하고 절감하는 기능입니다.

4개의 탭

📊 현황
이번 달 비용, 지난 달 대비 변화, 캐시 히트율, 절감 금액을 카드로 표시합니다. 에이전트별 비용 순위도 확인 가능합니다.
🔮 예측
캐시 TTL, 모델 티어, 스킬 예산을 슬라이더로 조절하면 예상 비용이 실시간으로 계산됩니다. (What-if 시뮬레이터)
💡 추천
AI가 분석한 비용 절감 추천 사항이 나열됩니다. 현재 티어 → 추천 티어 변경 시 예상 절감액을 보여줍니다.
📑 로그
개별 토큰 사용 기록입니다. 시간, 에이전트명, 토큰 수, 비용을 확인할 수 있습니다.

3-Gate 비용 최적화 원리

Metis.AI는 AI 호출을 3단계로 자동 최적화합니다:

Gate 1: 캐시
이전에 같은 질문이면 캐시에서 즉시 응답 → Gate 2: 라우터
간단한 질문은 저렴한 모델로 자동 라우팅 → Gate 3: 패커
프롬프트를 압축하여 토큰 절약

🚀 15. 릴리스 관리

사이드바의 릴리스 허브에서 접속합니다. AI 모델이나 워크플로우의 새 버전을 안전하게 배포하기 위한 기능입니다.

3가지 안전 장치

🔁
Replay 테스트
과거 실행 데이터를 새 버전에 다시 돌려봅니다.

결과:
PASS 통과
FAIL 실패
REGRESSION 성능 저하
👁
Shadow 테스트
실제 운영 트래픽을 새 버전에 조용히 보내서 결과만 비교합니다. 실제 사용자에게는 영향 없음.
🐦
Canary 배포
트래픽을 조금씩 새 버전으로 이동시킵니다.

5% → 25% → 50% → 100%

문제가 발견되면 자동으로 롤백됩니다.

Canary 배포 진행 과정

Replay 통과 → Shadow 검증 → Canary 5% → Gate 평가 → 트래픽 증가 → 100% 승격

❓ 16. 자주 묻는 질문 (FAQ)

Q. 워크플로우를 만들었는데 실행 버튼이 비활성화돼요.

노드가 하나도 없으면 실행 버튼이 비활성화됩니다. 프롬프트를 입력하고 "워크플로우 생성" 버튼을 먼저 클릭하세요.

Q. 저장 버튼을 눌렀는데 "저장 실패"라고 나와요.

서버 API가 연결되지 않은 상태일 수 있습니다. Docker 컨테이너(PostgreSQL, Redis)가 실행 중인지 확인하고, pnpm dev로 서버를 재시작해보세요. 서버가 연결되지 않아도 브라우저의 로컬 저장소에 자동으로 백업 저장됩니다.

Q. 워크플로우 이름을 바꿨는데 마켓에 새로 생성돼요.

캔버스 상단의 "이름 적용" 버튼은 이름만 변경합니다. 실제 저장은 우측 상단의 저장 버튼을 눌러야 합니다. 같은 이름이 이미 있으면 업데이트되고, 새 이름이면 새로 생성됩니다.

Q. 보고서가 깨져서 나와요.

보고서 파서가 데이터를 제대로 인식하지 못할 때 발생할 수 있습니다. AI Processing 노드의 프롬프트를 더 구체적으로 작성하거나, 모의해킹 노드의 경우 소스코드 파일이 올바르게 업로드되었는지 확인하세요.

Q. 이전에 만든 워크플로우가 사라졌어요.

빌더 화면은 세션 스토리지에 자동 저장되지만, 브라우저를 완전히 종료하면 사라질 수 있습니다. 중요한 워크플로우는 반드시 우측 상단 저장 버튼으로 서버에 저장하세요.

Q. FinOps에서 비용이 0원으로 나와요.

아직 워크플로우를 실행하지 않았거나, FinOps 로그 데이터가 쌓이지 않은 상태입니다. 워크플로우를 실행하면 토큰 사용 기록이 자동으로 누적됩니다.

Q. 특정 메뉴가 보이지 않아요.

로그인한 계정의 역할(Role)에 따라 접근 가능한 메뉴가 제한됩니다. 필요한 메뉴가 보이지 않으면 테넌트 관리자에게 역할 변경을 요청하세요.
