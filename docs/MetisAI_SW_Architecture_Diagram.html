Korean NLP Strategies

Strategy Trigger Example
Sequential Split 해서, 하면, 한 후, 하고 "로딩하면 분석해서 보내줘" → 3 phrases
Strategy C (Domain Noun) 및, 과/와 + DOMAIN_NOUN_MAP "보안취약점과 모의해킹" → 2 nodes
Strategy A (Verb Split) 및, 그리고 + all have verbs "검색 및 분석" → 2 nodes
Strategy B (Shared Verb) 및 + trailing verb "A 및 B를 점검" → "A 점검" + "B 점검"
Delivery Extraction Domain noun + embedded verb "취약점을 메일로 받을수" → node + email

BH-4: Structural Validator

Validation Rule Check Auto-Repair Action Severity
Trigger Node 워크플로우에 시작 트리거가 있는지 add-trigger blocking
Exit Node 종료 노드가 하나 이상 존재하는지 add-exit-node blocking
Retry Coverage 외부 호출 노드에 재시도 설정이 있는지 inject-retry warning
Audit/Monitoring 감사 로그/모니터링 노드 포함 여부 add-monitor warning
Branch Condition 분기 노드에 조건식이 설정되었는지 set-branch-predicate blocking
Branch Paths 분기 노드에 최소 2개 경로가 있는지 add-branch-paths blocking
Approval Gates 위험 노드 앞에 승인 게이트가 있는지 add-approval warning
Connector Valid 사용된 커넥터가 유효한지 replace-connector warning
8 Repair Types: add-trigger, add-exit-node, inject-retry, add-monitor, set-branch-predicate, add-branch-paths, add-approval, replace-connector. 각 repair action은 자동 적용 가능하며, 적용 후 re-validate가 수행됩니다.

BH-7: Simulation Engine (10-50 Samples)

Dynamic Sample Count
