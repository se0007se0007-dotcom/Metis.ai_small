Metis.AI 사용자 매뉴얼

AI 에이전트를 등록하고, 실행하고, 비용·품질·보안·이상을 한 화면에서 확인하는 방법을 단계별로 안내합니다. 현재 버전 기준입니다.

대상: 관리자 · 운영자 · 검토자 | 기준: 현재 빌드 | 접속: http://localhost:3000
1. 시작하기 2. 역할·권한 3. 대시보드 4. 기준정보 5. Agent 등록 6. 실행 7. 이력·4게이트·구간 8. 거버넌스 9. 설치·기동 10. 트러블슈팅 FAQ

1시작하기

웹 브라우저에서 http://localhost:3000 으로 접속해 로그인합니다. 메뉴는 업무 흐름 순서(보기 → 실행 → 기준정보 → 거버넌스)로 구성되어 있습니다.

① 무엇을 보나

홈 대시보드에서 전체 실행·비용·품질·이상을 한눈에.

② 무엇을 하나

에이전트 화면에서 실행하고 결과(4게이트)를 확인.

③ 무엇을 관리하나

기준정보에서 Agent·모델단가·사용자·운영기준값 편집.

처음이라면 추천 순서: 기준정보에서 Agent 확인 → 전용 실행화면 URL 설정 → 실행 → 실행 이력에서 4게이트·구간 확인.

2역할 · 권한 (RBAC)

계정은 테넌트(조직) 내 역할을 가집니다. 역할에 따라 보이는 메뉴와 가능한 작업이 다릅니다.

역할할 수 있는 일
PLATFORM_ADMIN플랫폼 전체 관리(테넌트 생성 포함), 모든 기준정보 편집.
TENANT_ADMIN소속 조직의 기준정보(Agent·사용자·팀·운영기준값) 편집, 게시 승인.
OPERATOR에이전트 실행, 실행 이력·대시보드 조회.
DEVELOPER빌더로 워크플로우 설계·실행, 커넥터 연결.
AUDITOR감사 로그·거버넌스 조회(읽기 중심).
VIEWER대시보드·이력 조회(읽기 전용).
권한 안내: 기준정보의 Agent 편집·전용 실행화면 URL 저장은 TENANT_ADMIN/PLATFORM_ADMIN 권한이 필요합니다. 저장이 안 되면 먼저 역할을 확인하세요.

3대시보드 보는 법

메뉴 홈. 최근 기간의 실행을 집계해 KPI·헬스·ROI로 보여줍니다.

상단 KPI

지표의미
Agent 총 수행기간 내 총 실행 건수. 클릭 시 전체 실행 이력 목록이 열립니다.
성공률 / 평균·최대 지연성공 비율과 응답 지연(ms). 지연이 큰 실행을 가려냅니다.
합계 비용기간 내 LLM 비용 합계($). 모델단가 기준정보로 계산.
헬스 / 등급운영기준값(임계치)로 산출한 상태(정상/주의/위험)와 등급(A~D).

구성 요소

  • 전체/성공/실패 실행 이력 — KPI 타일 클릭으로 목록 팝업. 행을 누르면 실행 상세로.
  • APM X‑View — 가로=시각, 세로=지연(ms) 산점도. 영역 드래그로 구간 선택해 상세 확인.
  • 메인/서브 에이전트 현황 — 어느 에이전트(또는 노드 구간)가 문제인지 한눈에.
상단의 메인 Agent / Sub‑Agent 필터를 고르면 대시보드 전체가 그 범위로 좁혀집니다. 필터 해제로 되돌립니다.

4기준정보(Master Data) 관리

관리자 메뉴 기준정보 아래에서 플랫폼 전반이 참조하는 값을 직접 편집합니다. 여기서 바꾸면 모든 화면(대시보드·실행·드롭다운)에 반영됩니다.

메뉴관리 대상
기준정보 › Agent메인/서브 에이전트 목록. 이름·코드·설명, 전용 실행화면 URL(launchUrl), 게시 여부. 서브→메인 승격(다중 메인 금지).
기준정보 › 모델 단가gpt/claude 등 모델별 입력/출력 단가. 비용 계산·모델 드롭다운의 원천.
기준정보 › 운영기준값시급(hourlyRate)·월 가동시간·헬스 임계치·등급 컷오프. 대시보드 ROI/헬스/등급 산출 기준.
기준정보 › 테넌트/팀 · 사용자조직(테넌트)·팀 계층, 사용자 등록/삭제.
관리 › Ingest 키외부 시스템이 SDK로 실행 결과를 보고할 때 쓰는 mts_… 키 발급/조회.

모델 단가가 화면에 반영되는 방식

등록 화면 등의 모델 선택 드롭다운은 하드코딩이 아니라 모델 단가 기준정보를 불러옵니다. 단가를 추가/수정하면 드롭다운과 비용 계산에 즉시 반영됩니다.

5Agent 등록 · 편집

메뉴 기준정보 › Agent. 에이전트의 이름·코드, 그리고 전용 실행화면 URL을 설정합니다.

  1. Agent 목록 열기 — 메인 에이전트와 그 아래 서브(노드)가 함께 표시됩니다. 표시 이름은 [코드] 이름 표준 형식.
  2. 편집 — 대상 Agent의 편집을 누르고 이름/코드/설명을 정리합니다.
  3. 전용 실행화면 URL 입력 — 외부 전용화면이 있는 에이전트는 전용 실행 화면 URL (선택)에 주소를 입력합니다(예: http://localhost:8600).
  4. 저장 — 저장 후 성공 토스트에 저장된 URL이 표시됩니다. 이 URL이 보이면 정상 저장된 것입니다.
  5. 게시 — 카탈로그에 노출하려면 게시(또는 ORB 승인)로 listed를 켭니다.
저장 확인 팁: 저장 시 토스트에 “실행화면 URL: …”이 뜨면 DB에 저장된 것입니다. “API가 옛 빌드일 수 있어요”가 뜨면 API 재시작이 필요합니다(아래 9·10장 참고).
서브 → 메인 승격은 지원되지만, 하나의 서브가 여러 메인에 매핑되는 것은 금지됩니다(중복 시 안내 후 차단).

6Agent 실행

메뉴 에이전트 › 운영/개발 또는 카테고리 화면에서 실행합니다. 실행은 어떤 경로든 대시보드·실행 이력·4게이트에 기록됩니다.

6‑1. 외부 전용화면 에이전트 (예: 테스트 자동화)

  1. 대상 Agent의 실행을 누르면, 등록해 둔 launchUrl 화면이 metis 안에 그대로 임베드되어 열립니다.
  2. 그 화면에서 평소처럼 입력하고 실행합니다(예: 코드 붙여넣기 → 실행).
  3. 실행이 끝나면 결과(보고서·비용·지연·구간)가 자동으로 metis에 기록되고, “기록되었습니다” 토스트가 뜹니다.
  4. 대시보드/실행 이력/AI 활동 로그에서 방금 실행을 확인합니다.
처음 한 번: 외부 화면(test‑agent) 컨테이너는 코드를 이미지에 포함하므로, 화면/기록 기능이 바뀌면 컨테이너를 다시 빌드해야 반영됩니다. (9장 참고)

6‑2. 사내 워크플로우 에이전트

빌더로 만든 워크플로우는 노드(서브 에이전트)가 순차 실행되고, 노드마다 4게이트가 평가되어 실행 상세에 노드별로 남습니다. 저장된 에이전트는 빌더로 가지 않고 key로 즉시 실행할 수 있습니다.

7실행 이력 · 4게이트 · 구간별 소요

실행 이력에서 행을 클릭하면 실행 상세가 열립니다. 여기서 한 실행의 모든 것을 봅니다.

실행 상세 구성

영역내용
4축 요약품질 · 보안 · 비용 · 수행시간을 점수/등급으로 요약.
구간별 소요실행을 구간(예: 분석 / LLM 리뷰 / 집계·기록)으로 나눠 막대 + 초·%로 표시. 가장 큰 구간을 강조해 병목을 바로 보여주고 개선 힌트를 제공.
품질(Sub‑Agent별)노드별 종합·등급·정확도·환각률.
보안 · 이상 · 알림위협/유출 카운트, 이상 탐지, 관련 FDS 알림과 적용 정책.

4‑게이트란

품질

정확도·환각률·응답품질로 결과의 신뢰도를 평가.

비용

토큰·모델단가로 실행 비용을 산출, 효율 등급화.

보안

입력 위협·출력 유출·도구체인 위험을 점검.

이상

평소와 다른 패턴(이상 이벤트)을 탐지.

AI 활동 로그

메뉴 에이전트 › AI 활동 로그. 누가·언제·무엇을 실행/변경했는지 감사 기록(AuditLog)을 보여줍니다. 에이전트 실행도 실행(EXECUTE) 항목으로 남습니다.

구간별 소요는 어디서 보나: 실행 이력 → 실행 클릭 → 실행 상세의 “구간별 소요” 패널. 외부 에이전트도 실행 화면이 구간 타이밍을 보고하면 구간이 표시됩니다.

8거버넌스 · 인사이트

메뉴용도
거버넌스 › 정책실행을 통제하는 규칙(비용 상한, 인증 요구, 감사 등). 위반 시 차단/경고.
거버넌스 › ORB에이전트 게시 심사. 승인되면 카탈로그에 노출.
거버넌스 › 효과성·KPI절감·ROI·효과 지표. 운영기준값(시급 등)을 반영해 산출.
거버넌스 › 런타임실행 중 위험 의사결정(차단·격리·승인요구)과 자동조치 이력.
인사이트 › 이상/평가기이상 탐지·품질 평가 결과를 집중적으로 분석.

9설치 · 기동

로컬 전체 스택은 start-metis.bat 한 번으로 기동합니다(7단계). PostgreSQL·Redis·FinOps 컨테이너 → DB 셋업 → 앱 기동 순.

  1. 프로젝트 폴더로 이동: cd /d "…\metis간소화버전"
  2. start-metis.bat 실행. 브라우저에서 http://localhost:3000 접속.
  3. 외부 화면(test‑agent) 기능을 갱신했다면 컨테이너 재빌드:
    docker compose -f finops-platform\docker-compose.yml up -d --build test-agent
  4. API 코드만 갱신했다면: pnpm --filter @metis/api build 후 API 재시작.
중요 — 데이터 보존: 시드(데모 데이터 주입)는 최초 1회만 실행됩니다. 이후 재시작에는 시드를 건너뛰어 저장한 Agent 편집(launchUrl)과 실제 실행 이력이 보존됩니다. (마커 파일 .metis_seeded 기준)
데모 데이터를 처음부터 다시 깔고 싶을 때만 .metis_seeded 파일을 삭제하고 재기동하세요. (이때 데모 데이터는 재생성되지만, 사용자의 실제 실행 이력은 보존되도록 시드가 보강되어 있습니다.)

10트러블슈팅

전용 실행화면 URL을 저장했는데 다시 열면 비어 있어요
① 저장 시 토스트에 URL이 표시되는지 확인 → 표시되면 정상. ② 안 되면 API가 옛 빌드일 수 있으니 pnpm --filter @metis/api build 후 API 재시작. ③ 권한(TENANT_ADMIN 이상)인지 확인.
재시작했더니 실행 이력·URL이 사라졌어요
과거에는 재기동 시 시드가 데이터를 덮어썼습니다. 현재는 시드가 최초 1회만 돌도록 바뀌어 보존됩니다. 그래도 사라지면 .metis_seeded 파일이 있는지 확인하세요(있어야 시드를 건너뜁니다).
실행은 되는데 대시보드/이력에 안 보여요
외부 화면 컨테이너가 옛 버전일 수 있습니다. test‑agent를 재빌드(--build)하고 브라우저를 Ctrl+F5 강력 새로고침 후 다시 실행하세요. 실행 후 “기록되었습니다” 토스트가 뜨는지 확인.
실행이 오래 걸려요 — 어디가 느린지 보고 싶어요
실행 상세의 구간별 소요 패널에서 구간별 시간·비중을 확인하세요. 외부 화면 로그(docker compose … logs -f --timestamps test-agent)의 [TIMING] 줄로도 구간을 볼 수 있습니다. (LLM 호출이 보통 가장 큼 — 병렬화/토큰 조정으로 단축)
기동 창이 바로 닫혀요
전제(Node·pnpm·Docker)가 설치되어 있는지, Docker Desktop이 실행 중인지 확인하세요. PostgreSQL이 뜰 때까지 스크립트가 대기합니다.

＋자주 묻는 질문

“에이전트”는 어떻게 등록되나요?
메인 에이전트는 워크플로우로, 서브 에이전트는 그 노드로 표현됩니다. 기준정보 › Agent에서 이름·코드·전용화면 URL을 편집합니다.
외부에서 만든 에이전트도 metis에서 관리되나요?
네. 전용 실행화면 URL을 등록하면 metis 안에 그 화면이 임베드되어 실행되고, 결과가 대시보드·이력·4게이트·AI 활동 로그에 기록됩니다. 서버‑투‑서버라면 Ingest 키로 보고할 수도 있습니다.
비용은 어떻게 계산되나요?
모델 단가 기준정보 × 토큰 사용량으로 산출되어 실행별로 기록되고, 대시보드에서 합산됩니다.
대시보드 등급/헬스 기준을 바꾸려면?
기준정보 › 운영기준값에서 임계치·등급 컷오프·시급 등을 편집하면 모든 집계에 반영됩니다.
본 매뉴얼은 현재 빌드 기준으로 작성되었습니다. 화면 구성이나 메뉴 명칭은 버전에 따라 일부 다를 수 있습니다.
