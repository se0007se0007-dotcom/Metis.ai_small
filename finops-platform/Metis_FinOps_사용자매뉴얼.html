Metis FinOps — 사용자 매뉴얼

ktds OPS.AI / Metis.AI(AgentOps) 플랫폼용 Agent FinOps 솔루션의 전 화면 사용 안내서입니다. 각 화면의 목적, 모든 지표·그래프·표 컬럼의 의미와 읽는 법을 상세히 설명합니다.

0. 솔루션 소개와 동작 원리

Metis FinOps는 모든 LLM 에이전트 호출이 거쳐 가는 OpenAI 호환 게이트웨이와, 비용을 기록·통제·최적화하는 컨트롤 플레인으로 구성된 벤더 중립 FinOps 플랫폼입니다. 에이전트는 코드 수정 없이 OpenAI 베이스 URL만 게이트웨이로 바꾸면 연동됩니다.

5계층 아키텍처

계층역할
L1 수집게이트웨이가 모든 호출을 가로채 토큰 5종(input/output/cache_read/cache_write/reasoning)을 분리 계측
L2 원장tenant→project→agent→run→step 귀속 차원의 비용 원장(SQLite), 모델별 단가 마스터
L3 통제예산 3단계, run 서킷브레이커, 루프 감지, 품질 게이트, 거버넌스 융합
L4 최적화counterfactual 절감 회계(캐시·라우팅·스킬패커), 예측·What-if·권고
L5 경험페르소나별 대시보드 8종 + 통합 테스트 에이전트

화면 공통 사용법

  • 왼쪽 사이드바에서 화면(페르소나)을 전환합니다. 데이터는 5초마다 자동 갱신됩니다.
  • 화면 좌하단의 연결 점(●)이 초록이면 컨트롤 플레인 정상 연결, 회색이면 끊김(재시도 중)입니다.
  • 대부분의 차트 우측 상단에 기간/그룹 선택 드롭다운이 있어 보고 싶은 범위로 바꿀 수 있습니다.
  • 금액은 USD 기준이며, 토큰은 K/M 단위로 축약 표시됩니다.

전 테넌트의 실시간 지출·절감·통제 현황을 한눈에 보는 관제 화면입니다. 운영자가 가장 먼저 보는 메인 대시보드입니다.

지출 추이 (좌측 누적 막대)

시간 버킷별 지출을 누적 막대로 보여줍니다. 우측 상단에서 기간(30분~7일)과 그룹 기준(테넌트별/에이전트별/모델별/프로바이더별)을 선택합니다. 버킷 크기는 기간에 따라 자동 조정됩니다(예: 1시간→1분, 24시간→15분).

모델별 지출 비중 (우측 도넛)

선택 기간 동안 어떤 모델이 비용을 많이 썼는지 비율로 표시합니다. 조각 위 % 라벨과 마우스 오버 시 정확한 금액·비율을 봅니다.

실시간 알림 피드

서킷브레이커·예산·거버넌스·이상감지 등 모든 통제 이벤트가 시간순으로 흐릅니다. 좌측 색 점은 심각도(critical warning info)입니다.

절감 구성 패널 (도넛 + 표)

절감액이 어떤 메커니즘에서 나왔는지 분해합니다. 표에는 메커니즘별 절감액·비중·건수와, 하단 합계에 캐시 히트율이 표시됩니다.

메커니즘의미
시맨틱 캐시동일/유사 질문의 응답을 재사용해 LLM 호출 자체를 회피(실비용 ~0). 회피된 원 비용 전액이 절감.
프롬프트(prefix) 캐시벤더 프롬프트 캐싱으로 cache_read 토큰이 일반 input 단가보다 싸게 과금된 차액.
3티어 라우팅 강등상위 모델 요청을 저가 모델로 처리했을 때, 원 모델로 처리했을 경우 대비 차액.
스킬패커툴 스키마를 전부 보내지 않고 필요한 것만 동적 로딩해 절약한 input 토큰의 비용.

2. 에이전트 정책 (레지스트리)

에이전트별 최적화 정책을 중앙에서 관리하는 화면입니다. 표의 배지를 클릭하면 즉시 토글됩니다(별도 저장 불필요).

컬럼의미
에이전트 / 설명레지스트리에 등록된 에이전트 이름과 용도. 미등록 에이전트는 첫 호출 시 자동 등록되며 기본값은 캐시 OFF(안전 기본값).
시맨틱 캐시이 에이전트의 응답 캐시 사용 여부. ✓ 적용 (TTL) 또는 미적용. 클릭으로 전환. 캐시 적합 워크로드(반복 FAQ)만 켭니다.
주력 모델 → 강등 대상최근 가장 많이 쓴 모델과, 강등 시 내려갈 대상 모델.
품질 게이트강등 대상 모델의 최근 7일 품질 실적 판정: 승인(샘플 충분+품질 기준 이상→강등 적용) · 카나리(샘플 부족→설정 비율만 강등하며 데이터 수집) · 보류(품질 미달→원 모델 유지). 배지에 평균 품질과 샘플 수가 표시됩니다.
강등라우팅 강등 활성/비활성. 비활성이면 예산 압박이 와도 강등하지 않습니다.
복잡도 라우팅활성 (≤0.3 강등) 시, 단순한 프롬프트(복잡도 점수 0.3 이하)가 상위 모델을 요청하면 경제 모델로 자동 강등 후보화. 품질 게이트·거버넌스 방어를 통과해야 실제 적용됩니다.
툴 레지스트리이 에이전트의 전체 툴 스키마 토큰 수(스킬패커 절감 계산의 기준값).
run 한도run당 최대 비용과 최대 스텝 수(서킷브레이커 기준).
24h Run / 품질최근 24시간 run 수와 평균 품질 점수.

품질 게이트가 중요한 이유

"싸니까 무조건 강등"이 아니라, 대상 모델이 실제로 품질을 유지하는지 검증한 뒤에만 강등합니다. 데이터가 부족하면 소수 카나리로만 시도해 안전하게 실측 데이터를 모읍니다.

3. 개발자 뷰

내 에이전트의 run 단위 비용 구조를 분석합니다. 어느 스텝·모델·토큰이 비쌌는지 추적합니다.

최근 Run 표

상단 드롭다운으로 에이전트를 필터링합니다. 각 행: Run ID, 에이전트, 상태(성공/차단/실패/실행중), 스텝 수, 토큰, 비용, 품질. 행을 클릭하면 아래 워터폴이 열립니다.

토큰 구성 (가로 막대)

선택한 run의 토큰을 5종(input/output/cache_read/cache_write/reasoning)으로 분해합니다. reasoning(사고) 토큰은 output 단가로 과금됩니다.

Run 비용 워터폴

run의 각 스텝을 비용 막대로 나열합니다. 막대 색·배지로 특이사항을 표시: 캐시(캐시 히트로 비용 0) · 강등(모델 다운시프트, 원모델→대상모델 표기) · reasoning(사고 토큰 발생). 어느 스텝이 run 비용을 끌어올렸는지 한눈에 보입니다.

Tip

테스트 에이전트 화면에서 "개발자 뷰에서 이 run 비용 보기" 버튼을 누르면 해당 run의 워터폴로 바로 이동합니다.

4. 운영 뷰

예산·롱테일 비용·서킷브레이커·GPU 풀을 감시하는 운영자 화면입니다.

일 예산 현황 (3단계 게이지)

상단 드롭다운(테넌트별/에이전트별/전체)으로 그룹을 선택합니다. 각 게이지의 막대 위 눈금이 3단계 임계입니다.

단계동작
소프트캡도달 시 알림만 발생(차단·강등 없음). 경고 신호.
강등이 임계 초과 시 해당 스코프의 호출을 저가 모델로 다운시프트(품질 게이트 통과 시).
하드컷도달 시 추가 요청을 차단(HTTP 429). 예산 초과 방어선.

막대 색: 정상(초록)/소프트 도달(노랑)/강등 도달(빨강). 행 우측에 현재 지출 / 하드 한도와 세 임계값이 표기됩니다.

에이전트별 Run 비용 분포 (p50/p95/p99)

에이전트별 run 비용의 백분위수를 막대로 비교합니다. p99가 p50보다 현저히 높으면(롱테일) 폭주 루프·과도한 툴 호출을 의심합니다. 정렬은 p99 내림차순.

백분위수(percentile) 읽는 법

p50=중앙값(절반의 run이 이보다 쌈), p95=상위 5% 경계, p99=상위 1% 경계. p99만 튀면 "대부분 싸지만 가끔 매우 비싼 run"이 있다는 뜻입니다.

통제 이벤트 로그

서킷브레이커·예산·구성변경·이상감지·가드레일 이벤트만 모아 봅니다.

셀프호스트 GPU 풀 (H100×4)

노드별 GPU 사용률·MEM·KV캐시 사용률·큐 깊이·시간당 비용을 표시합니다. 상단의 유휴비용/시간은 노드별 시간당비용 × (1−GPU사용률) 합으로, 놀고 있는 GPU에 새는 돈입니다. 하단 라인 차트는 노드별 사용률 추이(30분).

차단·실패 Run 표

최근 killed run의 사유·스텝·소진 비용. 어떤 통제가 왜 발동했는지 사후 분석합니다.

5. 재무 뷰

테넌트 쇼백·절감 회계·단위경제·모델 단가·FOCUS export를 다루는 재무/회계 화면입니다.

FOCUS 1.4 CSV 내보내기

버튼 클릭 시 FinOps Foundation FOCUS 1.4 호환 CSV를 받습니다. 표준 컬럼(ListCost=미적용가정, ContractedCost/BilledCost/EffectiveCost=실비용) 외에 x_ 접두 확장 컬럼으로 토큰 5종·절감액·데이터등급·리스크·캐시판정을 함께 제공해 회계/차지백 시스템에 연동합니다.

테넌트 쇼백 표

테넌트별 비용·절감·토큰·호출 수·점유율(전체 비용 중 비중). 어느 조직이 비용을 많이 쓰는지, 차지백 배분의 근거가 됩니다.

절감 회계 분해 (도넛)

개요의 절감 구성과 동일한 4종 분해 + 캐시 히트율·절감 합계. counterfactual(반사실) 회계 — "그 기능이 없었다면 들었을 비용" 대비 실제 비용의 차이를 절감으로 잡습니다.

단위경제: Cost-of-Pass 표

컬럼의미
Run집계 대상 run 수.
성공률품질 게이트 통과(pass) 비율. 80% 미만이면 빨강 경고.
평균 품질품질 점수 평균(0~1).
p50 비용run 비용 중앙값.
Cost-of-Pass성공 1건당 실질 비용. 낮을수록 비용 효율적.

품질-비용 폐루프 (이중축 라인)

시간에 따른 평균 품질(좌축)과 run당 평균 비용(우축)을 겹쳐 봅니다. 빨간 세로 점선은 구성 변경 이벤트(예: 라우팅 전환) 시점입니다. 변경 후 비용이 떨어지면서 품질도 함께 떨어졌는지 즉시 확인 — "비용을 아꼈더니 품질이 망가졌다"를 잡는 화면입니다.

모델 단가 마스터

모델별 단가(Input/Output/CacheRead/CacheWrite, $/1M 토큰)·티어·강등 대상을 표로 보여주며, 값을 직접 수정하고 저장하면 즉시 과금 계산에 반영됩니다(코드 수정 불필요). 단가 인상/계약가 변경 시 여기서 관리합니다.

비용 vs 절감 추이 (누적 막대)

5분 버킷으로 실비용과 절감액(회피 비용)을 누적해, 시간대별 효율을 봅니다.

6. 거버넌스 융합 (Patent 3 — 차별화 핵심)

비용 최적화가 보안·정책을 침해하지 않도록 캐시 재사용과 모델 라우팅을 데이터 등급·리스크 점수·정책 해시로 통제합니다.

상단 KPI

지표의미
정책 준수율민감 데이터의 캐시 누출이 0이면 100%. 누출이 생기면 하락 — 반드시 100% 유지가 목표.
캐시 정책 차단민감(PII/기밀) 또는 고위험 요청이라 캐시 재사용을 막은 건수.
리스크 강등 방어예산 압박에도 고위험 작업을 약한 모델로 떨어뜨리지 않고 안전 티어로 상향/유지한 건수(escalation).
거버넌스 적용 호출거버넌스 판정을 거친 전체 호출 수.

데이터 등급별 호출·비용 / 거버넌스 정책 / 이벤트 로그

데이터 등급별 호출 수·비용 분포를 막대로 보고, 현재 거버넌스 정책(민감 등급 목록, 고위험 차단 임계, 강등 방어 임계, 안전 최소 티어)을 확인·변경합니다. 우측 로그는 캐시 차단·리스크 상향 이벤트.

왜 차별화인가

일반 FinOps 도구는 비용만 봅니다. Metis는 캐시·강등 결정에 데이터 등급·리스크·정책 해시를 결합해, "비용 아끼다 민감정보가 캐시로 새거나 고위험 작업이 약한 모델로 떨어지는" 사고를 원천 차단합니다. 정책 해시가 바뀌면 기존 캐시는 자동 무효화되고, 캐시 키는 테넌트 스코프로 격리됩니다.

7. 예측 · 이상감지 (지능 계층)

월말 비용 예측, What-if 시뮬레이션, 자동 권고, 이상감지 5종, 품질 회귀 가드레일을 제공합니다.

What-if 시뮬레이션

정책을 바꾸면 월 비용이 얼마나 달라지는지 원장 실데이터 기반으로 추정합니다. 3개 레버를 조절하고 실행:

  • 캐시 TTL 배수 — 캐시 유지 시간을 늘렸을 때의 추가 절감(수확 체감 반영).
  • 강등 가능 호출 전면 강등 — 강등 가능한 모든 호출을 한 단계 내렸을 때의 절감 상한.
  • 툴 스키마 추가 압축 — 스킬패커 압축률을 더 높였을 때의 절감.

결과는 월 환산 기준선 → 시나리오 비용과 메커니즘별 절감 분해로 표시됩니다. 상한 추정이며 품질 게이트 통과를 가정합니다.

자동 권고

원장 분석으로 생성된 절감/안정화 제안 목록입니다. 각 권고에 예상 절감액이 붙고, [적용] 버튼으로 즉시 정책에 반영(+감사 로그 기록), [무시]로 닫습니다. 예: "강등 활성화", "복잡도 라우팅 제안", "캐시 후보", "run 한도 하향", "예산 검토".

품질 회귀 가드레일

강등된 run의 품질이 비강등 대비 일정 비율(기본 10%) 이상 떨어지면 자동 감지합니다. 자동 원복(METIS_AUTO_REVERT)이 켜져 있으면 해당 에이전트의 강등을 즉시 비활성화하고 알림을 남깁니다. 표에서 비강등 품질·강등 품질·하락률·조치(자동 원복됨 또는 수동 원복 버튼)를 확인합니다.

8. 인사이트

원장 데이터에서 자동 생성된 권고 카드 모음입니다. 별도 조작 없이 읽기만 하면 됩니다.

각 카드는 아이콘·제목·본문과 심각도 색(critical/warning/info)으로 구성됩니다. 다루는 주제:

  • 지출 집중 — 특정 테넌트가 지출의 40% 이상을 차지할 때.
  • 롱테일 비용 — p99가 p50의 8배 이상인 에이전트.
  • cost-of-pass 악화 — 성공률 80% 미만으로 성공당 비용이 비싼 구성.
  • 절감 회계 — 최근 24시간 절감 성과 요약.
  • 🔮 월말 비용 전망 — 예측 요약(상세는 7번 화면).
  • 이상감지 — 상위 이상 징후.
  • GPU 유휴/증설 검토 — 사용률 기준 권고.
  • 구성변경 영향 — 품질-비용 폐루프 분석.
  • 🛡️ 거버넌스 융합 — 차단·방어·누출 준수 현황.

9. 테스트 에이전트 (통합 데모)

실제 동작하는 데모 에이전트입니다. 코드를 올리면 분석·테스트·LLM 리뷰를 수행하고, 그 비용/절감/품질이 즉시 다른 화면에 반영됩니다.

사용 순서

  1. Python/Java/C 소스를 붙여넣거나 파일 선택(또는 샘플 버튼). 언어는 자동 감지됩니다.
  2. 기능 테스트 실행 클릭 → ① 정적 분석 ② 컴파일/격리 실행 ③ LLM 코드 리뷰 3스텝(요약→리스크→권고, 게이트웨이 경유).
  3. 결과 보고서를 화면에서 확인하고 Word(.docx, 차트·표 포함) 또는 Markdown으로 내려받습니다.
  4. 상단 메타(Run ID·언어·소요·LLM 비용·모드)와 "개발자 뷰에서 이 run 비용 보기"로 비용 추적.

주의

동적 테스트 단계에서 코드가 로컬(격리 프로세스, 시간 제한)에서 컴파일·실행됩니다. 신뢰할 수 있는 코드만 사용하세요. JDK/gcc 미설치 환경에서는 정적 분석만 수행됩니다(우아한 저하).

C. 자주 묻는 질문

Q. 데이터가 안 보이거나 "연결 끊김"이 떠요.

좌하단 연결 점이 회색이면 컨트롤 플레인(:8500)이 응답하지 않는 상태입니다. 서비스 기동 여부를 확인하세요. 화면을 바꿔도 자동 재시도합니다.

Q. 정책을 바꿨는데 반영이 안 돼요.

에이전트 정책·거버넌스·단가는 즉시 반영됩니다. 서버 로직/스키마를 변경한 경우에는 서비스를 재시작해야 메모리의 옛 코드가 교체됩니다.

Q. 강등됐는데 품질이 떨어지면요?

품질 회귀 가드레일(7번 화면)이 자동 감지하고, 자동 원복이 켜져 있으면 강등을 즉시 비활성화합니다.

Q. 캐시가 동작하지 않는 에이전트가 있어요.

시맨틱 캐시는 중앙 레지스트리에서 에이전트별로 켜야 합니다(안전 기본값 OFF). 또한 민감 데이터·고위험 요청은 거버넌스가 캐시를 의도적으로 차단합니다.
