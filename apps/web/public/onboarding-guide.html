Agent 연동 온보딩 가이드

여러분이 개발한 Agent를 Metis에 연결하면, 호출마다 품질·보안·비용·이상 4게이트로 자동 평가·통제되어 대시보드와 이력에 한 화면으로 모입니다. 이 가이드 한 장이면 연동이 끝납니다.

대상: Agent를 개발·운영하는 모든 구성원 · 소요: 약 10분 · Python SDK 기준
STEP 1
기준정보 등록
내 Agent를 Metis에 등록
STEP 2
Ingest 키 발급
팀/Agent용 인증 키 받기
STEP 3
표준 스니펫 계측
코드에 데코레이터 한 줄
STEP 4
대시보드 확인
4게이트 현황·이력 확인
딱 두 가지만 하면 됩니다 — ① 기준정보에 Agent 등록, ② 코드에 표준 스니펫 한 줄. 평가·집계·통제는 플랫폼이 자동으로 합니다. 정답·근거 데이터가 없어도 4게이트는 모두 동작합니다.

1 기준정보에 Agent 등록

메뉴: 기준정보 > Agent. 내 Agent를 먼저 등록해 두면 표시명·비용·ROI·전용 실행화면·정책이 붙습니다.

등록 시 정해 두는 값(아래 스니펫에서 그대로 사용합니다):

항목설명스니펫 인자
Agent 이름(코드)대시보드·이력에서 이 Agent를 식별하는 이름. [코드] 이름 표준 표기agent
메인 Agent 키여러 단계로 구성되면 묶음 기준. 단일이면 이름과 같게workflow_key
사용 모델호출 모델 ID. 모델 단가 기준정보에 등록된 ID와 일치해야 비용·₩이 정확model
ROI 설정(선택)시급·수작업 분 → 절감액/순가치 계산에 사용(운영기준값과 연동)—
전용 실행화면 URL(선택)Metis 안에서 이 Agent를 바로 실행/열람—
미등록도 기록은 됩니다. 다만 표시명·ROI·실행화면·정책이 빠진 “느슨한 Agent”로 잡혀 한눈에 구분됩니다. 정식 거버넌스 대상이 되려면 등록을 권장합니다.

2 Ingest 키 발급

메뉴: 관리 > Ingest 키(또는 “Agent 연동” 모달). 외부 Agent가 Metis로 실행을 보고할 때 쓰는 인증 키입니다.

  • 팀/Agent 단위로 발급하세요. 누가 무엇을 보고하는지 책임 추적이 됩니다.
  • 생성 시 평문 키(mts_live_…)는 딱 한 번 표시됩니다 — 그때 복사해 안전한 곳(시크릿 매니저)에 보관.
  • 키에 agentName 허용목록을 걸면, 등록되지 않은 이름으로 보고하는 스푸핑을 차단합니다(권장).
키는 비밀입니다. 코드·깃에 하드코딩하지 말고 환경변수/시크릿으로 주입하세요.

3 표준 스니펫으로 계측

실제 Agent 함수 위에 데코레이터 한 줄. 필수 4가지만 채우면 4게이트가 모두 평가됩니다.

from metis import Metis

m = Metis(
    api_key="mts_live_…",   # 팀/Agent별 Ingest 키 (환경변수 권장)
    base_url="https://<사내 Metis API>",  # 내 Metis API 서버 (로컬은 http://localhost:4000)
)

# ── 표준 연동 (이 형태로 통일) ───────────────────────────
#   필수 : agent, question_arg(입력), return값(출력), model
#   권장 : workflow_key(메인) / step_key(단계)
#   → 위 입력만으로 품질·보안·비용·이상 4게이트가 모두 평가됨
@m.eval(
    agent="<코드>-<이름>",          # 기준정보에 등록한 Agent 이름과 동일
    workflow_key="<메인 Agent 키>",  # 대시보드 그룹핑(서브→메인 합산)
    step_key="main",                # 단계가 여러 개면 단계명으로 구분
    task_type="qa",                 # qa / summarize / classify ...
    model="claude-haiku-4-5",       # 모델 단가 기준정보의 모델 ID와 일치
    question_arg="q",               # 함수의 입력 kwarg 이름과 동일
)
def my_agent(q):
    # ★ LLM "원본 응답 객체"를 그대로 return → 출력·토큰 자동 추출(비용 계산)
    return call_my_real_llm(q)

my_agent(q="환불 정책 알려줘")
토큰이 0이면 비용도 0 입니다. 함수가 LLM 원본 응답 객체(OpenAI/Anthropic/LangChain)를 그대로 return하면 텍스트·토큰이 자동 추출됩니다. 문자열만 반환해야 한다면 토큰을 직접 넘기세요: m.log_run(agent=…, input=…, output=…, model=…, tokensIn=…, tokensOut=…)

인자 → 무엇에 쓰이나

구분인자역할
필수agent이 실행이 귀속될 Agent 이름(기준정보와 일치)
필수question_arg입력 인자 이름 → 품질·보안(입력) 게이트
필수return값출력 → 품질·보안(출력)·LLM 심판
필수model모델 ID → 비용 게이트(단가로 ₩ 계산)
권장workflow_key / step_key메인 Agent / 단계(Sub-Agent) 구분 → 대시보드 롤업
선택context_argRAG 근거 문서(런타임 검색 결과) → 환각 탐지 강화
선택ground_truth_arg정답(평가셋) → 정확도 30/70 보정. 오프라인 평가 때만

(선택) RAG Agent면 한 줄 추가

이미 검색해 둔 근거 문서를 docs로 같이 넘기면, “답이 근거에 실제로 있는가”를 대조해 환각을 잡아냅니다.

@m.eval(..., context_arg="docs")
def my_agent(q, docs=None):
    return call_my_real_llm(system=f"참고 문서:\n{docs}", user=q)

docs = retriever.search(q)          # 질문마다 동적으로 검색
my_agent(q=q, docs=docs)            # 검색 결과를 kwarg로 전달

※ ground_truth_arg는 정답이 있는 평가/회귀 테스트에서만 사용합니다. 운영 트래픽은 보통 생략하며, 이때 정확도는 LLM 심판이 100% 채점합니다.

4 대시보드·이력에서 확인

호출 즉시 평가·거버넌스를 거쳐 집계됩니다. 같은 workflow_key는 메인=서브 합계로 롤업됩니다.

품질Quality 40%
정확도·환각·응답품질. LLM 심판(Claude)로 채점. 근거/정답 있으면 더 엄밀.
보안Security 30%
규칙 기반(LLM 비호출). 입력 인젝션 + 출력 PII·시크릿 유출 탐지.
비용Cost 15%
모델 단가 × 토큰 → ₩ 환산. 효율·절감 추정.
이상Anomaly 15%
지연·비용·토큰 추세 이탈 탐지(자동).

확인 위치:

  • 홈 대시보드 — Agent별 품질·보안·비용·이상, Health 등급, ROI·순가치 한눈에
  • 실행 이력 — 호출 단건별 입력/출력/게이트 점수/₩/지연, 행 클릭 시 상세
  • 품질평가(Insights) — 게이트별 분포, LLM 심판 사용·모델·비용
정답을 주지 않아도 품질은 LLM 심판으로, 보안은 규칙으로 채점되므로, 팀이 달라도 모든 Agent가 동일 잣대로 통제·비교됩니다. 이것이 거버넌스 플랫폼의 핵심 가치입니다.

운영·거버넌스 규칙 (권장 표준)

  • 등록을 기본 절차로 — 새 Agent는 기준정보 등록 후 배포. 미등록은 “느슨한 Agent”로 노출되어 자연히 등록을 유도.
  • 키는 팀/Agent별 + allowlist — 책임 추적 + 스푸핑 차단.
  • 모델 ID 일치 강제 — 비용·FinOps 정확도 확보(모델 단가 기준정보와 동일 ID).
  • 정책 설정 — 허용 도구/도메인·런타임 게이트로 위반 시 자동 차단·다운그레이드 + 알림(모니터링을 넘어 통제).
  • 단계가 여러 개인 Agent — 같은 workflow_key 아래 step_key를 단계별로 다르게 → Sub-Agent 단위 추적.

배포 전 체크리스트

  • 기준정보에 Agent 등록 완료(이름·메인키·모델)
  • 팀/Agent용 Ingest 키 발급 + 시크릿으로 주입(하드코딩 금지)
  • 표준 스니펫의 필수 4인자(agent·question_arg·return값·model) 채움
  • question_arg 값이 함수 kwarg 이름과 일치
  • 함수가 LLM 원본 응답을 return(또는 토큰 직접 전달)
  • 호출 1건 보낸 뒤 대시보드/이력에 표시되는지 확인

자주 묻는 질문

base_url에는 뭘 넣나요?

metis.ai 같은 외부 도메인이 아니라 우리 Metis API 서버 주소입니다. 로컬은 http://localhost:4000, 운영은 사내 Metis API 주소.

Agent 이름을 기준정보에 꼭 등록해야 하나요?

연동·기록만 목적이면 필수는 아닙니다(자동 기록됨). 단, ROI·실행화면·정책 등 거버넌스 기능을 쓰려면 같은 이름으로 등록하세요. 키에 allowlist를 걸었다면 이름이 목록과 일치해야 합니다.

비용이 0으로 나와요.

토큰이 0이기 때문입니다. LLM 원본 응답 객체를 return하거나, model과 함께 tokensIn/tokensOut를 직접 넘기세요.

정답(ground_truth)을 매번 어떻게 주죠?

운영에선 안 줍니다. 정답이 있는 평가/회귀 테스트 데이터셋을 돌릴 때만 사용합니다.
